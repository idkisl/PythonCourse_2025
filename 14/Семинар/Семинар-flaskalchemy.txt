Задание.
    Создать веб-приложение для учета книг в библиотеке с тремя моделями.

Модели:
    1. Автор (Author)

    - Имя (обязательное)
    - Год рождения (необязательное)
    - Страна (необязательное)
    Связь: один автор → много книг

    2. Книга (Book)

    - Название (обязательное)
    - Автор (внешний ключ к Author)
    - Год издания (необязательное)
    - Жанр (необязательное)
    - Доступна? (по умолчанию да)
    Связь: одна книга → много выдач

    3. Выдача (Loan)

    - Книга (внешний ключ к Book)
    - Читатель (имя, обязательное)
    - Дата выдачи (автоматически)
    - Дата возврата (необязательное)


Функционал (веб-интерфейс):

    Главная страница (/)
        - Список всех книг с их статусом
        - Кнопки: "Добавить книгу", "Все авторы", "Выданные книги"

    Страница авторов (/authors)
        - Список всех авторов
        - Форма добавления нового автора


    Страница книг (/books)
        - Список всех книг в таблице
        - Возможность фильтровать по доступности
        - Кнопка "Добавить книгу"

    Добавление книги (/books/add)
        - Форма с полями: название, выбор автора, год, жанр
        - Если автора нет в списке — ссылка "добавить автора"

    Выдача книги (/loan/<book_id>)
        - Форма с именем читателя
        - Меняет статус книги на "выдана"
        - Создает запись в таблице Loan

    Возврат книги (/return/<loan_id>)
        - Кнопка "вернуть книгу"
        - Меняет статус книги на "доступна"
        - Заполняет дату возврата в записи Loan

    Поиск (/search)
        - Поиск книг по названию или автору
